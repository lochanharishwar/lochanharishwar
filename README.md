## How I Think About Software

Most tools are built around features. **I build around outcomes.**

The question I ask before writing any code is: *what does the person on the other side of this interface actually need?* Not what they typed into a search bar ‚ÄĒ **what they're actually trying to accomplish.**

That philosophy shows up in everything I ship:

- **SYSTEMATIC-SEARCH** doesn't just retrieve ‚ÄĒ it **profiles, synthesizes, and personalizes**
- **GOALBOOST** doesn't just track ‚ÄĒ it **nudges, suggests, and adapts**
- **A-STARLIGHT** doesn't just explain ‚ÄĒ it **responds to exactly where you are** in the learning journey

**Good software is empathetic.** It does the cognitive work so the user doesn't have to.
